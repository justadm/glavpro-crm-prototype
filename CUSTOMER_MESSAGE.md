# Передача результата (для заказчика)

## Что сделано
- Прототип CRM-воронки на Joomla в виде компонента `com_glavpro_crm`.
- Карточка компании: стадия, доступные действия, инструкция/скрипт, история событий.
- Ограничения по стадиям: менеджер не может выполнить действия “вперёд”, пока не выполнены обязательные шаги.
- Журнал событий компании (event log) + вычисление/обновление стадии по правилам.

## Как посмотреть демо
1. Откройте список компаний (фронтенд CRM).
2. Откройте карточку компании.
3. Последовательно выполняйте доступные действия — стадия будет двигаться только при выполнении обязательных шагов.

## Доступ
- Фронтенд CRM закрыт для гостей: требуется авторизация в Joomla (логин/пароль предоставляются отдельно).

## Развёртывание
- Самый простой путь: Docker (см. `deploy/README.md`).
- Компонент устанавливается как ZIP, собирается командой `bash scripts/build_component_zip.sh` (архив: `dist/com_glavpro_crm.zip`).

## Что можно улучшить в боевой версии
- Настроить ACL Joomla (права на действия по ролям).
- Добавить отображение деталей событий (payload) в истории.
- Добавить интеграционные тесты (контроллеры/маршруты/DB) и e2e сценарии.
